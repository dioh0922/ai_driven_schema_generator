# V2 要件定義 (@Definition)

作成日: 2026-06-05  
ステータス: **草案 (Draft)**

## 1. V2 のゴール

> **既存のスキーマファイル（SQL/Prisma）を解析し、構造の可視化（ER図）と、それに基づいたさらなるスキーマ拡張を可能にする。**

V1 の「新規生成」に加え、V2 では「既存資産の活用と可視化」を軸に、開発者の設計補助ツールとしての価値を高める。

---

## 2. スコープ一覧

### V2 の主要機能 (Must)

| ID | 機能 | 説明 |
|----|------|------|
| V2-REV-IN | ファイルアップロード | 既存の `schema.sql` または `schema.prisma` をブラウザからアップロード |
| V2-REV-PARSE | リバース解析 | アップロードされたファイルをパースし、内部メタデータ（エンティティ・リレーション）を抽出 |
| V2-ER-GEN | ER図生成 | 解析されたメタデータから Mermaid 形式の `ER_diagram.md` を生成 |
| V2-UI-ER | ER図プレビュー | Mermaid.js 等を使用し、ブラウザ上で ER 図を動的に描画・表示 |
| V2-EXT-GEN | 既存ベースの生成 | 解析した構造を前提に、「テーブルの追加」や「カラムの変更」を AI に依頼 |

### V2 で検討する項目 (Should/Could)

| ID | 機能 | 優先度 | 説明 |
|----|------|----|------|
| V2-UI-EDIT | 生成結果の微修正 | Should | UI 上で直接コードを編集し、再保存・ER 反映 |
| V2-MULTI-FILE | 複数ファイル対応 | Could | 複数の SQL ファイルを跨いだ依存関係の解析 |
| V2-DOC-GEN | ドキュメント生成 | Could | テーブル・カラムの説明を AI が補足したデータ辞書の生成 |

---

## 3. ユーザーストーリー (V2)

1. 利用者は手元にある `schema.sql` を画面にドラッグ＆ドロップする。
2. アプリが即座にテーブル間のリレーションを解析し、画面に ER 図を表示する。
3. 利用者は「これに『通知設定』テーブルを追加して」とチャット形式（または追加要望欄）で入力する。
4. AI は既存のスキーマを考慮した上で、新しい `schema.sql` と `schema.prisma` を生成し、ER 図も更新される。

---

## 4. 技術的要件

### パース戦略
- SQL (PostgreSQL) および Prisma Schema DSL のパーサー実装、またはライブラリの選定。
- AI にパース結果をコンテキストとして渡すための「中間表現（JSON）」の定義。

### 可視化
- Mermaid.js を用いたフロントエンドでのレンダリング。
- `output/ER_diagram.md` への書き出し。

---

## 5. 受け入れ条件 (V2)

- [ ] `.sql` / `.prisma` ファイルをアップロードして読み込めること。
- [ ] 読み込んだファイルの内容に基づいて、正しいリレーションを持つ Mermaid ER 図が表示されること。
- [ ] 既存の構造を維持したまま、新しいテーブルを追加生成できること。
- [ ] 生成された `ER_diagram.md` をダウンロードできること。
