# リファクタリング計画 (2026-06-05)

作成者: Gemini CLI (@Inverter & @Realistic & @Structural)

## 1. 概要
V1 のカオス検証および品質レビューに基づき、信頼性と保守性を向上させるためのリファクタリングを実施する。

## 2. 短期タスク (V1 リリース前必須)
これらの修正は、システムの致命的な欠陥（ファイル破損、DoS、パース失敗）を回避するために今日中に実施する。

### 🚨 [優先度: 高] 並列アクセス時のファイル破損対策
- **現状**: 全リクエストが `output/schema.sql` を共有。
- **修正**: リクエストごとに一意の ID (UUID または Timestamp) を生成し、`output/[ID]/` ディレクトリに保存するように変更。

### 🚨 [優先度: 高] 入力バリデーションの導入
- **現状**: プロンプトの文字数制限がなく、巨大なリクエストでサーバーがハングする恐れ。
- **修正**: Fastify のスキーマバリデーションを使用し、`prompt` の最大文字数を 2000 文字に制限する。

### 🚨 [優先度: 中] 静的配信の標準化
- **現状**: `fs.readFile` による自前配信。
- **修正**: `@fastify/static` を導入し、セキュリティとパフォーマンス（キャッシュ制御）を向上させる。

## 3. 中長期タスク (V1.1 / V2 向け)
コードの可読性とメンテナンス性を向上させるためのリファクタリング。

### [設計方針] エラーハンドリングの構造化
- 独自の `AppError` クラスを作成し、エラーメッセージと適切な HTTP ステータスコードをカプセル化する。

### [設計方針] プロンプトの外部化
- `src/core/generator.ts` に埋め込まれているプロンプトテンプレートを、専用の `prompts/` ディレクトリまたは設定ファイルに分離する。

### [ディレクトリ構成]
```text
src/
  api/
    schemas/     # JSON Schema 定義
    errors/      # カスタムエラークラス
  core/
    generator.ts # 生成ロジックのみ
    prompts.ts   # プロンプト管理
  index.ts       # プラグイン登録とルート定義
```

## 4. 実行スケジュール
1. **Step 1**: `@fastify/static` の導入と `index.ts` のクリーンアップ。
2. **Step 2**: スキーマバリデーションの追加。
3. **Step 3**: `generateSchema` の出力をディレクトリ分割型に変更。
4. **Step 4**: 全体動作確認（カオステストの再実行）。
