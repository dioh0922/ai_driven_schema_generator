## 背景

現状は `POST /generate` による API のみ。V1 では **画面から要望を入力し、`schema.sql` / `schema.prisma` を取得できる** ことを最優先とする。

リバースエンジニアリング・ER 図は **V1 対象外**（[v1_release.md](./v1_release.md) 参照）。

## [詳細な要件内容]

### 1. ユーザーストーリー（V1）

- **利用者として**、画面に要望を書いて送信したい
- **利用者として**、生成された `schema.sql` と `schema.prisma` をダウンロードしたい
- **利用者として**、エラー時に何が悪いか分かるメッセージを見たい

### 2. 画面構成（V1 最小）

| 画面要素 | V1 | 説明 |
|----------|----|------|
| 要望入力エリア | Must | 複数行テキスト |
| 生成ボタン | Must | 未入力時は無効またはエラー |
| ローディング表示 | Must | 二重送信防止 |
| 結果プレビュー（SQL / Prisma） | Must | タブまたは2ペイン表示 |
| 個別ダウンロード | Must | 各ファイルに DL ボタン |
| ZIP 一括 DL | Should | 任意 |
| ER 図プレビュー | — | **V1 では実装しない** |

### 3. 入出力仕様

#### 入力

```json
{ "prompt": "要望テキスト（必須、1〜8000文字程度）" }
```

#### 出力（V1 で提供するファイル）

| ファイル名 | V1 |
|------------|-----|
| `schema.sql` | Must |
| `schema.prisma` | Must |
| `ER_diagram.md` | **対象外（V2）** |

#### API レスポンス（V1 推奨形式）

```json
{
  "message": "Schema generated successfully",
  "files": [
    { "name": "schema.sql", "content": "..." },
    { "name": "schema.prisma", "content": "..." }
  ]
}
```

### 4. 非機能要件（V1）

| 区分 | 要件 |
|------|------|
| 応答時間 | AI 生成中はローディング表示（数十秒想定） |
| エラー | 空 prompt・API キー未設定・AI 失敗を日本語表示 |
| セキュリティ | `GEMINI_API_KEY` はサーバーのみ |
| 配信 | Fastify + `public/` 静的配信、単一ポート |

### 5. 受け入れ条件（V1）

- [ ] 画面で要望を入力し「生成」できる
- [ ] SQL と Prisma を画面で読める
- [ ] 両ファイルをローカルに保存できる
- [ ] 空入力でエラーが分かる
- [ ] API キー未設定等で失敗理由が分かる
