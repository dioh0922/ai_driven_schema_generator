# V1 カオス検証報告書 (@Inverter)

作成日: 2026-06-05  
検証者: Gemini CLI (@Inverter)

> **「このシステムは必ずどこかで壊れる」**

V1 の実装を徹底的にレビューし、システムをクラッシュさせたり、予期せぬ挙動を引き起こしたりするための「攻撃ポイント」を特定した。

---

## 🚨 リスク指摘

### 1. 巨大入力によるリソース枯渇 (DoS)
- **場所**: `POST /generate`
- **内容**: ユーザーが数メガバイトの `prompt` を送りつけた場合、Fastify のデフォルト設定や AI モデルのトークン上限によりサーバーがハング、または高額な API 費用が発生する可能性がある。現状、入力文字数制限（バリデーション）が一切ない。

### 2. AI 応答の脆弱なパース
- **場所**: `src/core/generator.ts`
- **内容**: `text.split('---')` でパースしているが、AI が気まぐれに `---` を余計に出力したり、逆に一つも出力しなかった場合、即座に 500 エラーを返す。また、AI が「承知しました」などの余計な挨拶を含めると、そのままファイルに書き込まれ、構文エラーを引き起こす。

### 3. 書き込み競合によるファイル破損
- **場所**: `src/core/generator.ts` (`fs.writeFile`)
- **内容**: 複数のユーザーが同時に `/generate` を叩いた場合、全員が同じ `output/schema.sql` に書き込もうとする。結果としてファイルが途中で混ざったり、不完全な状態でダウンロードされたりする。V1 は「単一ユーザー」を想定しすぎている。

### 4. ディレクトリトラバーサルの潜在リスク
- **場所**: `src/index.ts` (`servePublic`)
- **内容**: `path.join(PUBLIC_DIR, filename)` を使用しており、ルート定義で `/styles.css` など個別に定義しているため現在は安全だが、将来的にワイルドカード（`/*`）で配信するように変更した場合、`../../.env` などを読み取られる危険がある。

---

## 🛠 推奨される防御策

1. **入力バリデーションの導入**:
   - プロンプトの最大文字数（例: 2000文字）を制限する。
2. **パース処理の堅牢化**:
   - AI に対して「JSON で返せ」と指示するか、正規表現でより厳密にコードブロックを抽出する。
3. **ユーザー/リクエストごとの分離**:
   - `output/` 直下ではなく、リクエストごとに `output/[timestamp]-[uuid]/` のようなディレクトリを作成して保存する。
4. **Fastify 公式プラグインの利用**:
   - 手書きの `servePublic` ではなく、`@fastify/static` を使用して安全な静的配信を行う。

---

## 💣 結論
V1 は「ハッピーパス（正常系）」では動くが、悪意のある入力や並列アクセスに対しては非常に脆い。リリース前に少なくとも「入力文字数制限」と「書き込み競合対策」を行うことを強く推奨する。
